# RelayForge SDK Contract

The backend OpenAPI document is the source of truth:
`../relay-forge-server/openapi/relayforge.yaml`.

Regenerate SDK path types with:

```bash
npm run generate:api
npm run check:api-contract
```

## Current Strategy

The SDK intentionally keeps stable handwritten public method names instead of doing a large
generated-client rewrite. Drift prevention comes from generated OpenAPI path types plus typed path
builders:

- `src/generated/relayforge.ts` is generated by `openapi-typescript`.
- `src/paths.ts` exports `API_PATHS` and `buildPath`.
- `API_PATHS` must satisfy the generated OpenAPI path key union.
- `src/client.ts` public methods must build routes with `buildPath(API_PATHS...)`.
- `src/__tests__/client.test.ts` captures SDK requests for auth, guild/channel, role/member,
  message, DM, admin, media, and voice paths.

## Adding Or Changing Routes

1. Add or update the backend route.
2. Update `relay-forge-server/openapi/relayforge.yaml`.
3. Run `npm run generate:api` from this repository.
4. Add or update the matching `API_PATHS` entry in `src/paths.ts`.
5. Route the SDK public method through `buildPath(API_PATHS...)`.
6. Add or update SDK path tests in `src/__tests__/client.test.ts`.
7. Run `npm run typecheck`, `npm test`, and `npm run check:api-contract`.

Avoid raw route strings in `src/client.ts`. The generic `get`, `post`, and `delete` helpers exist
for low-level escape hatches, but new first-class SDK routes should use the typed path builders.
